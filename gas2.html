はじめに：本ガイドラインの目的と背景

1. 当社の現状と目指す方向性

  • 現状認識： AI活用を全社方針とする一方、各部署には依然として非効率な手作業や定型業務が散在。
  • 課題： 属人的な作業によるミス、時間的コストの増大。
  • 目指す姿： 全社員が自律的に業務改善を推進する文化を醸成し、高付加価値業務へシフトする。

2. GAS（Google Apps Script）の位置づけ

  • Google Workspaceの機能を拡張し、自動化・連携を実現するプログラミング環境。
  • 特別な開発環境は不要で、Googleアカウントがあれば誰でも利用可能。

3. GAS活用の戦略的意義

  • 親和性： 日常業務で利用するツールを直接操作可能。
  • 費用対効果： 追加コストなく導入でき、迅速な効果が期待できる。
  • AI活用基盤の構築： データ収集・整形スキルの習得は、高度なAI活用のための必須要件。
GASスキルレベル定義と全体像

本ガイドラインでは、習熟度に応じて5段階のスキルレベルを定義し、体系的なスキルアップを支援します。個人の成長が組織の成長に繋がるモデルです。

全社GAS活用推進

  • Lv.1 ファーストタッチ：GASを一度は試したことがある
    └ 目的：GASの概念理解と成功体験の獲得
  • Lv.2 パーソナル活用：日常的にGASを活用し、自身の業務を効率化
    └ 目的：個人の反復作業を自動化し、生産性向上を実感
  • Lv.3 チーム貢献：GASを使い分け、チームの成果に貢献
    └ 目的：複数サービス連携を駆使し、チーム・部署のワークフローを改善
  • Lv.4 プロセス変革：内外のツールを組合せ、新たな価値を創造
    └ 目的：独自のツール開発により、再現性ある効率化や自動化を実現
  • Lv.5 全社DX推進：GASを軸に、組織全体の生産性向上を牽引
    └ 目的：GAS活用の標準化、後進育成、戦略立案を通じて事業に変革をもたらす

【Lv2-STEP1】状況に応じたコードを書いてみよう

テーマ：タスクリストのB列のステータスが「完了」になったら、その行を灰色にする

このコードは「編集されたセルの情報」をチェックし、もし条件に合えば行の色を変えます。

function onEdit(e) {
  // 編集されたセルの情報を取得
  const range = e.range;
  const sheet = range.getSheet();
  
  // B列が編集され、かつ値が「完了」の場合
  if (range.getColumn() == 2 && range.getValue() == '完了') {
    // 編集された行全体を取得して背景を灰色にする
    const row = range.getRow();
    sheet.getRange(row, 1, 1, sheet.getLastColumn()).setBackground('#dddddd');
  }
}
【Lv2-STEP2】コードを自分で修正してみよう

お題：「完了」なら灰色、もし「対応中」なら黄色にしてみましょう。

`else if` を使って条件を追加します。色も変えてみましょう。

function onEdit(e) {
  const range = e.range;
  const sheet = range.getSheet();
  const row = range.getRow();
  
  // B列が編集された場合のみ処理
  if (range.getColumn() == 2) {
    if (range.getValue() == '完了') {
      sheet.getRange(row, 1, 1, sheet.getLastColumn()).setBackground('#dddddd');
    } else if (range.getValue() == '対応中') {
      sheet.getRange(row, 1, 1, sheet.getLastColumn()).setBackground('#fffacd'); // 黄色
    }
  }
}

このように、条件や色を変えるだけで、自分のルールに合わせた自動化ができます。

【Lv2-STEP3】トリガーを設定しよう

`onEdit`は特別で、編集時に自動で動きます。ここでは別の例として「毎日朝9時に期限切れをチェックする」という時間主導のトリガーを設定してみます。

  1. スクリプトエディタの左メニューから、⏰ (時計) のアイコン [トリガー] をクリックします。
  2. 右下に表示される [+ トリガーを追加] ボタンをクリックします。
【Lv2-STEP4】トリガーの詳細を設定しよう

設定画面で「いつ」「何を」実行するかをGoogleに教えます。

  • 実行する関数を選択: `checkDeadline` （※事前にこの関数を作成しておく）
  • イベントのソースを選択: `時間主導型`
  • 時間ベースのトリガーのタイプを選択: `日付ベースのタイマー`
  • 時刻を選択: `午前8時～9時`

設定が完了したら [保存] をクリックします。

function createMinutes() {
  // テンプレートファイルをコピー
  const templateFile = DriveApp.getFileById('TEMPLATE_FILE_ID');
  const newFile = templateFile.makeCopy('議事録_' + Utilities.formatDate(new Date(), 'JST', 'yyyy/MM/dd'));
  
  // ドキュメントの内容を編集
  const doc = DocumentApp.openById(newFile.getId());
  const body = doc.getBody();
  body.replaceText('{{DATE}}', Utilities.formatDate(new Date(), 'JST', 'yyyy年MM月dd日'));
  body.replaceText('{{MEETING_NAME}}', '定例会議');
  doc.saveAndClose();
  
  // メール通知
  const emails = ['member1@company.com', 'member2@company.com'];
  GmailApp.sendEmail(
    emails.join(','),
    '議事録を作成しました',
    '議事録のリンク: ' + newFile.getUrl()
  );
}
IDのハードコーディングを避けよう
コード内に直接IDを書く（ハードコーディング）と、テンプレートファイルが変わっただけでコード修正が必要になります。上級者は、これらのIDをスクリプトプロパティや設定用シートに外出しして管理します。

【Lv3-STEP4】エラーハンドリングの導入

他人が使うツールは、予期せぬエラーに備える必要があります。

function createMinutesSafely() {
  try {
    // --- ここに、成功するのが当たり前の処理を書く ---
    createMinutes(); // メインの処理を呼び出す
    SpreadsheetApp.getUi().alert('議事録の作成と通知が完了しました。');

  } catch (e) {
    // --- もし try の中でエラーが起きたら、ここの処理が動く ---
    const errorMessage = `エラーが発生しました。\n\n詳細: ${e.message}\nファイル: ${e.fileName}\n行番号: ${e.lineNumber}`;
    Logger.log(errorMessage); // ログに詳細を記録
    SpreadsheetApp.getUi().alert(errorMessage); // ユーザーにエラーを通知
  }
}
`try...catch`構文は、プログラムの「保険」です。テンプレートファイルが削除されるなど、予期せぬ事態が起きてもプログラムが途中で止まらず、利用者に状況を伝えられます。
