GitHub無料プラン活用ガイド

社内ナレッジ共有のためのスタートアップマニュアル

はじめに：なぜ今、GitHubなのか？

まず、GitHub（ギットハブ）とは何か、一言でいうと「プログラムのソースコードや様々なファイルをインターネット上で保存・管理・共有できるサービス」です。

個人の記録

「あの時の状態に戻したい！」という時に、いつでも過去の状態にタイムスリップできます。

チームの連携

「誰が、いつ、どこを、何のために変更したか」が全て記録されるので、複数人での作業がとてもスムーズになります。

そして素晴らしいことに、個人や小規模チームであれば、無料プランでほとんどの主要機能が使えてしまいます。 それでは、具体的な中身を見ていきましょう。

第1章: GitHub無料プランで「できること」一覧

これが無料というのは本当に驚きです。無料プランの主な機能をリストアップしました。

機能カテゴリ 具体的にできること 解説
リポジトリ パブリック・プライベート共に無制限に作成可能 リポジトリとは、ファイルやフォルダを保管しておく場所（プロジェクトの置き場）のことです。公開用（パブリック）も、非公開用（プライベート）も、好きなだけ作れます。
共同作業 共同作業者の人数 無制限 プライベートなリポジトリでも、招待する共同作業者の数に制限はありません。チームでの開発に最適です。
自動化 (CI/CD) GitHub Actions (月2,000分まで) プログラムのテストやWebサイトの公開などを自動化する機能です。月2,000分（約33時間）も無料で使え、多くのプロジェクトでは十分な時間です。
パッケージ管理 GitHub Packages (500MBまで) 作成したプログラム部品（パッケージ）を保管しておく場所です。無料でも500MBのストレージが使えます。
Webサイト公開 GitHub Pages HTML/CSS/JavaScriptで作った静的なWebサイトを無料で公開できます。自分のポートフォリオ作成に最適です。
セキュリティ Dependabotによる脆弱性アラート プロジェクトが利用している外部プログラムの脆弱性を自動で通知し、修正を提案してくれます。
タスク管理 Issues, Projects 「バグ修正」「新機能の追加」といったタスクを管理し、誰が何を担当しているかが一目でわかります。

◆ 有料プランとの主な違いは？

無料プランでも非常に強力ですが、有料プラン（TeamやEnterprise）では、より高度な管理・セキュリティ機能が追加されます。

  • 高度なブランチ保護: 「部長の承認がないと重要なファイルは変更できない」といった、より厳密なルールを設定できます。
  • 必須レビュー担当者 (Code Owners): 特定のコードを変更する際に、自動で担当者をレビューワーに指定できます。
  • ActionsやPackagesの利用枠増加: 大規模なプロジェクトで無料枠を超えて利用する場合に必要になります。
  • SAMLシングルサインオン: 会社の認証システムと連携できます。

結論：まずは無料で始めて、チームが大規模になったり、より高度なセキュリティや管理が必要になったりしたタイミングで、有料プランを検討すれば全く問題ありません。

第2章: ゼロから始める！GitHubかんたん3ステップ

ステップ1：アカウントを作成しよう

  1. GitHub公式サイトにアクセスします。
  2. 「GitHubに登録する」または「サインアップ」ボタンを押し、メールアドレス、パスワード、ユーザー名を入力します。
  3. ユーザー名は後から変更できますが、あなたのIDとしてURLなどにも使われるので、よく考えて決めましょう。
  4. 画面の指示に従い、パズルなどで認証を済ませればアカウント作成は完了です。

ステップ2：リポジトリ（プロジェクトの置き場）を作成しよう

  1. ログイン後、画面右上の「+」アイコンから「New repository」を選択します。
  2. 以下の項目を設定します。
    • Repository name: リポジトリの名前（例: my-first-project）
    • Description: (任意) プロジェクトの簡単な説明
    • Public / Private: 社内用ならPrivateが安心です。
    • Add a README file: チェックを強く推奨します。
  3. 「Create repository」ボタンを押せば、プロジェクトの置き場所が完成です！

ステップ3：ファイルを置いてみよう（2つの方法）

作成したリポジトリにファイルを追加してみましょう。最初は簡単なWeb画面からの操作がおすすめです。

方法A：【一番かんたん】Web画面からアップロード

  1. リポジトリのページで「Add file」→「Upload files」を選択。
  2. ファイルをドラッグ＆ドロップするか選択。
  3. 画面下の「Commit changes」ボタンを押して完了！

方法B：【本格的】自分のPCからGitを使ってアップロード

チーム開発で必須となる方法です。「Git」というツールのインストールが必要です。

  1. git-scm.comからGitをインストール。
  2. PCでコマンドを実行してファイルをアップロードします。

方法Bの具体的なコマンドの流れ

PCで作業 → addで選択 → commitで記録 → pushでアップロード、という流れです。

# 1. リポジトリをPCにコピー (初回のみ)
git clone [リポジトリのURL]

# 2. ファイルの変更・追加を選択
git add .

# 3. 変更内容をメッセージ付きで記録
git commit -m "最初のファイルを追加"

# 4. GitHubにアップロード
git push

第3章: チーム作業の要！基本機能を使ってみよう

GitHubの真価はチームでの共同作業で発揮されます。ここでは最低限知っておきたい2つの機能を紹介します。

Issues (イシュー): タスク管理の掲示板

  • リポジトリの「Issues」タブから作成します。
  • 「〇〇のバグを修正する」「〇〇の機能を追加する」といったタスクを書き出します。
  • 担当者を割り当てたり、コメントで議論したりできます。
  • プロジェクトの「やることリスト」として活用できます。

Pull Request (プルリクエスト): 変更内容のレビュー依頼

  • 共同作業の中心となる、最も重要な機能です。
  • 自分の変更をいきなり本体に反映させず、「こういう変更をしましたが、レビューして取り込んでください」と依頼を出す仕組みです。
  • 他のメンバーが変更内容を確認し、問題なければ承認（Merge）することで、安全にコードを更新できます。

まとめ

ここまでの内容が、GitHubを無料で使い始めるための基本の全てです。

  • 無料でもリポジトリは無制限。個人・チーム開発の基本機能は全て揃っています。
  • まずはアカウントとPrivateリポジトリを作り、Web画面からファイルをアップロードしてみることから始めましょう。
  • 慣れてきたらGitをPCに導入し、clone → add → commit → push の流れをマスターしましょう。
  • Issuesでタスクを管理し、Pull Requestで安全な共同作業を実現しましょう。

これらの基本をまとめたナレッジを社内で共有し、実際に簡単なプロジェクト（例えば、社内ドキュメント管理など）で皆さんに触ってもらうのが、定着への一番の近道だと思います。
